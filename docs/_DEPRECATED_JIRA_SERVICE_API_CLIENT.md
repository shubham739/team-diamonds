# [DEPRECATED] Jira Service API Client (`jira-service-api-client`)

> **DEPRECATED ‚Äî This component is archived and no longer in active use.**
>
> Its only consumer, `jira-service-adapter`, is also deprecated (see [JIRA_SERVICE_ADAPTER.md](_DEPRECATED_JIRA_SERVICE_ADAPTER.md)). The production pipeline goes directly from `jira_service` to `jira_client_impl` without this layer. Both components are retained for reference only.

## Overview

`jira-service-api-client` was an auto-generated, type-safe HTTP client for `jira-service`. It was generated from the service's OpenAPI specification using [`openapi-python-client`](https://github.com/openapi-generators/openapi-python-client) and served as the network layer consumed by `jira-service-adapter`.

## Design

- Generated by `openapi-python-client` from the service's OpenAPI spec (`openapi.json` at the repo root).
- Built on `httpx` (sync and async support) with `attrs`-based model classes.
- Each endpoint has its own module under `jira_service_api_client/api/default/` with `sync()`, `sync_detailed()`, `asyncio()`, and `asyncio_detailed()` variants.
- Models live under `jira_service_api_client/models/` ‚Äî use `UNSET` (not `None`) for optional fields you want to omit from the request.
- `errors.UnexpectedStatus` is raised on unexpected HTTP status codes when `raise_on_unexpected_status=True`.

## Installation

This package is a workspace member ‚Äî it is installed automatically when you run `uv sync` from the repository root.

## Usage

### Unauthenticated (health check, public endpoints)

```python
from jira_service_api_client import Client
from jira_service_api_client.api.default import health_check_health_get

client = Client(base_url="https://baii6ilfl2.execute-api.us-east-2.amazonaws.com/prod")

with client as c:
    result = health_check_health_get.sync(client=c)
    print(result)
```

### Authenticated (issue CRUD)

```python
from jira_service_api_client import AuthenticatedClient
from jira_service_api_client.api.default import (
    list_issues_issues_get,
    get_issue_issues_issue_id_get,
    create_issue_issues_post,
    update_issue_issues_issue_id_put,
    delete_issue_issues_issue_id_delete,
)
from jira_service_api_client.models import CreateIssueRequest, UpdateIssueRequest, Status

client = AuthenticatedClient(
    base_url="https://baii6ilfl2.execute-api.us-east-2.amazonaws.com/prod",
    token="<atlassian-oauth2-token>",
)

with client as c:
    # List issues
    response = list_issues_issues_get.sync(client=c, status=Status.IN_PROGRESS, max_results=10)

    # Get a single issue
    issue = get_issue_issues_issue_id_get.sync(client=c, issue_id="PROJ-42")

    # Create an issue
    body = CreateIssueRequest(title="Fix the bug", desc="Details...", status=Status.TO_DO)
    new_issue = create_issue_issues_post.sync(client=c, body=body)

    # Update an issue (only supply fields you want to change)
    update_body = UpdateIssueRequest(status=Status.COMPLETED)
    updated = update_issue_issues_issue_id_put.sync(client=c, issue_id="PROJ-42", body=update_body)

    # Delete an issue
    delete_issue_issues_issue_id_delete.sync(client=c, issue_id="PROJ-42")
```

### Async usage

Every endpoint module also exposes `asyncio()` and `asyncio_detailed()` variants:

```python
async with client as c:
    result = await list_issues_issues_get.asyncio(client=c)
```

## Key Models

### `Status` (enum)

```python
class Status(str, Enum):
    TO_DO = "to_do"
    IN_PROGRESS = "in_progress"
    COMPLETED = "completed"
```

### `CreateIssueRequest` / `UpdateIssueRequest`

Both use `UNSET` (not `None`) for fields you want to omit:

```python
from jira_service_api_client.types import UNSET
from jira_service_api_client.models import CreateIssueRequest, Status

body = CreateIssueRequest(
    title="My issue",
    status=Status.TO_DO,
    # desc, members, due_date, board_id omitted ‚Üí sent as UNSET, not null
)
```

## Regenerating from OpenAPI Spec

This client is generated ‚Äî do not edit it by hand. To regenerate after updating the service:

```bash
make generate-client
```

This starts the service locally, fetches `/openapi.json`, and runs `openapi-python-client generate`. The output directory (`jira_service_api_client/`) is gitignored; the committed copy in `components/jira_service_api_client/` is the pinned snapshot used by the rest of the repo.
